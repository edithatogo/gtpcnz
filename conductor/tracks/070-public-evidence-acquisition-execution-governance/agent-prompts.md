Use these prompts only for future public evidence acquisition execution work under Track 070.

## WP-070-A governance-shell

Read Track 069 and the current public-source evidence docs. Write only governance material for execution readiness. ## WP-070-B custody-chain

Define the required custody chain for any future public source: canonical URL, retrieval timestamp, licence/access status, raw artefact path, SHA-256 checksum, transform script, processed artefact path, schema check, and validation-gate relationship. Keep missing elements as blockers.

## WP-070-C stop-conditions

List stop conditions for private administrative data, patient-level data, confidential OIA responses, stakeholder analysis, unpublished expert elicitation, unclear licence/access, missing checksums, missing raw files, failed transforms, and failed validation gates.

## WP-070-D claim-boundary

Check that all Track 070 text keeps `public_benchmark` / `calibration_readiness_only`. Do not describe readiness evidence, failed comparisons, or acquisition governance as calibrated model validation.
