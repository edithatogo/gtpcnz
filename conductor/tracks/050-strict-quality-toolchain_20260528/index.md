# Strict Quality Toolchain

This track captures the repo-quality upgrades that sit below the modelling work: static typing, dependency automation, coverage, edge-case testing, mutation testing, property-based testing, profiling, and GitHub enforcement.

## Files

- `spec.md` - current-state audit, target toolchain, and acceptance criteria.
- `plan.md` - implementation steps.
- `metadata.json` - track status and output contract.

## Boundary

This track changes quality gates and developer workflow. It must not change public model claims, Streamlit content, or calibration posture except to make those boundaries easier to verify.
