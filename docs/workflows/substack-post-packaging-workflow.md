# Substack post packaging workflow

Use this workflow before a post is marked Substack-ready.

## Steps

1. Confirm the post thesis and claim boundary.
2. Add or verify the thumbnail/hero-image spec.
3. Check every figure:
   - alt text;
   - native image caption, verified in the Substack draft JSON when the post is written by script;
   - source/status label;
   - caveat.
4. Check every source link:
   - funding type documents linked at first relevant mention;
   - policy sources linked near the claim;
   - end source list retained only as backup.
5. Check every table:
   - preserve if comparison matters;
   - convert to cards only if table is too wide;
   - avoid bullet conversion unless no comparison is lost.
6. Check Substack rich content:
   - dividers;
   - buttons/callouts;
   - embedded report/dashboard links;
   - formulae or LaTeX only where useful and explained.
7. Record a QA note listing thumbnail, figures, table decisions and source-link decisions.

## Done means

- The post can be pasted or scripted into Substack without losing captions, alt text intent, source navigation or table meaning. A separate italic paragraph near an image does not count as a native image caption.
