# PHOs: useful function or payment friction?

It is easy to turn any discussion about PHOs into a fight about whether they should exist.

That is the wrong starting point.

The better question is: what functions are we asking PHOs to perform?

Some functions may be valuable. Population health work matters. Quality improvement matters. Data support matters. Local relationships matter. Equity initiatives matter. Shared services can matter, especially for small practices.

But those functions are different from being the gateway through which public primary care funding must pass.

When an organisation performs both population-health functions and payment-intermediation functions, it becomes harder to know which parts are adding value and which parts are adding friction. A provider may need to navigate contracts, enrolment rules, pass-through arrangements and local processes before public subsidy reaches the clinical contact. That can make the system slower and less transparent.

The question should not be “abolish PHOs?”

The question should be:

> Which PHO functions add value, and which payment-gateway functions could be replaced by a national rules-based claims platform?

A national payment platform could allow accredited providers to claim for eligible contact types directly, while PHO or locality entities are funded separately for population health, equity, quality improvement and data support.

That would be cleaner. It would let us evaluate PHO functions on their own merits. It would also make it easier for new models to enter the market: nurse practitioner-led care, pharmacist care, kaupapa Maori providers, allied health, urgent care, ambulance treat-and-refer, hybrid rural services and integrated telehealth.

This is not an argument for removing accountability. It is an argument for making accountability clearer.
