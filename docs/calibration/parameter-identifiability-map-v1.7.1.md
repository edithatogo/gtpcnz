# Parameter identifiability map v1.7.1

This map separates observable, estimable, stakeholder-elicited and provisional parameters. It is designed to make the scaffold more credible and to show which assumptions can be tested first.

| Identifiability class | Count |
|---|---:|
| Partly observable; need and unmet need require triangulation | 8 |
| Estimable with variation; may require quasi-experiment | 4 |
| Partly observable; requires funding/payment documents and OIA/data access | 12 |
| Provisional / theoretical until data access | 10 |
| Estimable from provider/workforce and practice data | 11 |
| Detectable through anomaly analysis and audit; not directly observable ex ante | 6 |
| Stakeholder-elicited / MCDA-informed | 10 |
| Estimable from linked administrative data | 9 |

## Priority rule

The first empirical phase should focus on Tier 1 parameters that are either directly estimable from linked data or testable through clear natural experiments. Stakeholder-elicited and political-economy parameters should inform MCDA and risk registers, not headline predictive claims.
