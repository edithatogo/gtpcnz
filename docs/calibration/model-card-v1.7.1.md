# Model card v1.7.1 — full parameterised primary care funding scaffold

## Model name
Full parameterised scaffold for primary care funding architecture in Aotearoa New Zealand.

## Version
v1.7.1 hardening release. Builds on the v1.7.0 full parameterised scaffold.

## Intended use
- Compare the relative logic of funding architectures.
- Identify which assumptions are load-bearing.
- Support Substack explanation, policy discussion, stakeholder validation and OIA/data planning.
- Prepare the pathway for future empirical calibration.

## Not intended for
- Claiming precise reductions in emergency department presentations, admissions or costs.
- Producing an implementation business case.
- Replacing stakeholder validation, equity review or linked-data analysis.
- Claiming that any organisation has endorsed the model.

## Core policy question
Does a hybrid architecture — capitation for population responsibility plus an uncapped, scheduled, rules-based fee-for-service stream for eligible primary medical activity plus place-based accountability — allow lower-cost upstream care to expand safely before need becomes hospital demand?

## Inputs
The scaffold uses 70 parameters across demand, supply, funding, governance, ambulance, hospital, equity, risk and implementation domains. v1.7.1 tiers these into core, extended and exploratory parameters.

## Outputs
- Access and supply indices.
- Hospital pressure indices.
- Fiscal/gaming risk indices.
- Hybrid viability scores.
- Scenario comparison and sensitivity outputs.

## Validation status
Parameterised scaffold only. Not empirically calibrated to linked New Zealand administrative data.

## Known limitations
- Many parameters remain source-informed priors or placeholders.
- Some political-economy variables are better suited to stakeholder MCDA than quantitative prediction.
- PHO pass-through, urgent-care advice, and appropriation-transfer details require OIA/data access or formal stakeholder input.
- Māori, Pacific, rural and disability impacts require dedicated equity review before public recommendation posts are finalised.

## Key failure modes
1. Readers treat model indices as observed outcomes.
2. Uncapped fee-for-service is misread as uncontrolled fee-for-service.
3. Equity and place-based accountability are treated as optional safeguards rather than core design requirements.
4. The model is attacked as seventy assumptions rather than a tiered, falsifiable scaffold.
5. Public release outruns OIA responses or stakeholder validation.

## Interpretation rule
Use the model to ask better questions and prioritise validation. Do not use it to claim final effect sizes.
