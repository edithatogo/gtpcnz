# Diagram production skill

Use this skill when creating a conceptual diagram, causal map, pathway map, control stack or public argument map.

## Inputs

- Reader question.
- Claim boundary.
- Source/status label.
- Target surface: Substack, Quarto, GitHub Pages or Streamlit.
- Output size and format.

## Workflow

1. State the one thing the diagram must explain.
2. Choose the simplest visual grammar: flow, stack, matrix, loop, axis or timeline.
3. Draft the source diagram in Mermaid, Graphviz, SVG or Python.
4. Use plain labels. Avoid jargon unless the diagram defines it.
5. Add a status label: conceptual, toy, model-generated index, evidence readiness, calibration readiness or empirical.
6. Export the image in the required formats.
7. Write alt text that describes both the visible elements and the relationship shown.
8. Write a Substack caption and a Quarto caption.
9. Add the output to the figure manifest.

## Done means

- Source file exists.
- Exported image exists.
- Alt text exists.
- Caption exists.
- Status label exists.
- Caveat exists if conceptual or model-generated.
