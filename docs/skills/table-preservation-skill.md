# Table preservation skill

Use this skill when converting drafts for Substack, Quarto or GitHub Pages.

## Policy

Do not automatically flatten tables into bullet points.

Use tables when:

- the reader needs row/column comparison;
- scenarios, funding types, evidence status or validation requirements are compared;
- flattening would lose the structure of the argument.

Use cards when:

- the table is too wide for mobile;
- each row is a self-contained idea;
- the reader needs scan-friendly blocks.

Use bullets when:

- the items are short;
- order matters more than comparison;
- no row/column relationship is lost.

## Done means

- The conversion decision is documented.
- Tables that must remain tables are preserved.
- Converted tables have a card or bullet rationale.
